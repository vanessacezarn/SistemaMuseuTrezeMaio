# Sistema de Gerenciamento de Acervos para o Museu Treze de Maio
Trabalho final das disciplinas de Implementação de Banco de Dados e de Laboratório de Desenvolvimento de Software.

**Integrantes do grupo:** Éric Martins da Silva, Guilerme Frazzon Rolim, Luiza Lewandowsi Karlec e Vanessa Cezar do Nascimento
### ➤ Contexto e Justificativa do trabalho
O Museu Treze de Maio, dedicado à preservação da cultura afro-brasileira em Santa Maria, possui um importante patrimônio que precisa ser digitalizado, catalogado e disponibilizado. O museu necessita de soluções tecnológicas para gerenciar dois acervos com demandas distintas — o Acervo Histórico (jornais, atas, fotos, cartas, relatos, objetos etc.) e a Biblioteca Comunitária (livros, revistas e jornais). Para garantir a preservação desse material e ampliar o acesso ao público, é essencial desenvolver um sistema de gerenciamento eficiente e acessível.
### ➤ Objetivo Geral do trabalho
Projetar, desenvolver e implantar soluções de software integradas que permitam a catalogação, o gerenciamento e a consulta pública dos acervos do Museu Treze de Maio, utilizando o SQL Server como sistema gerenciador de banco de dados(SGBD).
### ➤ Fases do trabalho
#### 1) Análise e Prototipagem
É a fase em que ocorrem reuniões com os stakeholders, equipe do Museu, para o levantamento, análise e documentação detalhada dos requisitos. Nesta etapa, é necessário elaborar um documento de Especificação de Requisitos de Software (ERS), contendo os requisitos funcionais, os requisitos não funcionais, os diagramas de caso de uso e um protótipo focado na interface de usuário do sistema de acervo bibliográfico
#### 2) Modelagem e Projeto do Banco de dados
Projetar um modelo de dados flexível para o Acervo Histórico e estruturado para a Biblioteca, garantindo a integridade referencial e aplicando as formas normais quando aplicável. Nessa fase deve-se desenvolver um Diagrama Entidade-Relacionamento(DER) - modelo conceitual, Esquema Relacional- modelo lógico e um Dicionário de Dados. 
#### 3) Implementação(SQL) e Consultas. 
Implementar o banco de dados no SQL Server, popular o banco de dados com exemplos realistas e elaborar consultas complexas que simulem o uso real do sistema. Nesta etapa, será elaborado script SQL com os comandos de DDL - Data Definition Language, script SQL com os comandos DML - Data Manipulation Language e um relatório de consultas SQL.
#### 4) Desenvolvimento do Software e Testes
Desenvolver o software funcional para o registro, controle e consulta das obras da biblioteca e do acervo histórico. Deverá ser entregue ao final dessa fase um Sistema de Gestão da Biblioteca, com sofwtare funcional e código-fonte, e um relatório de testes e validação
#### 5) Implantação e Capacitação
Planejar a entrega do sistema, incluindo a instalação no local e a capacitação da equipe que utilizará as ferramentas. Assim, é necessário entregar um plano de implantação contendo a infraestrutura necessária e o cronograma de go-live, além de um manual do usuário e o material de treinamento para a equipe do museu.
